Interpretation

  • Torque: Consistent: F5/F1/F7/F9A use 'Average Surface Torque kN.m'. Outlier: F15C uses 'Surface Torque Average N.m' (different unit/name).
  • Hookload: Consistent: F5/F1/F7 use 'Average Hookload kkgf'. F9A has many hookload-related channels. Outlier: F15C uses 'Hookload N' variants (different unit).
  • RPM: Common pattern: many wells include 'Average Rotary Speed rpm'. Outlier: F15C uses c/s fields (e.g., Surface RPM c/s, Bit RPM - Total RPM c/s).
  • SPP: F5/F1/F7/F9A have standpipe pressure channels; F15C has no SPP match in this scan.
  • Flow in: F5/F1/F7/F9A: Mud Flow In L/min. Outlier: F15C uses m3/s and includes cementing flow channels.
  • Flow out: F5/F1/F7 have Mud Flow Out fields; F9A has no match here; F15C has several flow-out fields in m3/s / percent variants.
  • Pit volume: F5/F1/F7: not found. F9A: many 'MudPit Volume Average ... m3'. F15C: pit volume exists (m3) incl. Total Pit Volume etc.
  • Block position: F5/F1/F7/F9A: Block Position m. Outlier: F15C uses Block/Topdrive Position variants.
  • Bit depth: F5/F1/F7/F9A: bit depth fields exist. F15C: no bit depth match in this scan.
  • Hole depth: All wells have hole depth variants; naming differs slightly. F15C includes 'Hole Depth m' + mnemonic field.
  • Decision: Drop F15C_Statoil for the current pipeline due to missing SPP/Bit Depth and inconsistent units/naming across multiple core channels.
